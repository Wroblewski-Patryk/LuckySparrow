# Brand Personality Tokens

## Chosen Personality Tokens

- Primary: Warm Guidance
- Secondary: Calm Utility
- Secondary: Editorial Precision

## Canonical Brand Pairing

- Product brand: `Aviary`
- Header and logo family: `Cormorant Garamond`
- UI and body family: `Inter`
- Canonical logo asset: `docs/ux/assets/aviary-logomark.svg`
- Preview asset: `docs/ux/assets/aviary-logomark-preview.png`

## Typography Rules

- Use `Cormorant Garamond` for:
  - Aviary wordmark lockups
  - flagship headers
  - ceremonial section titles
- Use the logotype/header treatment as:
  - uppercase `AVIARY`
  - generous letter spacing
  - lighter weights such as `300` or `400`
  - spacious composition with visible breathing room
- Use `Inter` for:
  - paragraphs
  - labels
  - helper copy
  - operational UI
  - attribution or secondary signatures such as `by LuckySparrow`

## Naming Boundary

- `Aviary` is the first-party product and app-shell brand.
- The embodied digital personality currently has no fixed proper name.
- Do not hardcode a stable character name for the personality in product UI
  until that identity is intentionally formed from the personality itself.

## Translation Notes

- Typography:
  readable, welcoming, with editorial contrast between ceremonial headers and
  operational UI
- Color: restrained neutrals with humane accent moments
- Surfaces: soft but disciplined, low-noise, confidence-building
- Motion: gentle orientation and confirmation feedback
- Interaction tone: supportive, transparent, never manipulative

## Implementation Expectation

- First-party web surfaces should treat this pairing as the default brand
  contract unless a narrower canonical screen reference explicitly overrides a
  local detail.
