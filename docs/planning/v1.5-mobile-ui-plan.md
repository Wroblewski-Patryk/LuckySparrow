# v1.5 Mobile UI Plan

Last updated: 2026-05-12

## Purpose

`v1.5` is the native/mobile UI continuation after the verified `v1.1` web
responsive milestone. It uses the web screenshot learnings without copying the
web layout directly into a small screen.

## Scope

Included:

- Expo-managed React Native mobile UI
- conversation-first native shell
- shared app-facing `/app/*` contract visibility
- mobile-specific hierarchy, touch targets, spacing, and scroll behavior
- proof through TypeScript, Expo web export, and screenshot capture where
  device/simulator proof is not yet available

Excluded until a narrower task:

- final native auth transport
- offline sync
- push notifications
- provider secret entry
- internal debug/operator surfaces
- App Store or Play Store release automation

## Acceptance Criteria

For a v1.5 mobile UI slice to count as ready:

- TypeScript passes.
- The route renders through Expo web export or a native device/simulator proof.
- The first viewport prioritizes conversation or the current task, not admin
  contract inventory.
- Touch targets are at least visually large enough for mobile use.
- Mobile UI remains a thin client over backend-owned `/app/*` contracts.
- No provider secrets, internal debug surfaces, or local cognition logic are
  introduced.

## Next Checkpoints

1. Capture Expo Go or simulator proof for the home, chat, personality,
   settings, and tools routes when the runtime is available.
2. Decide whether the simple shared route rail should remain the v1.5
   navigation model or be replaced by native tabs after device proof.
3. Decide native auth transport posture before wiring app-facing data.
4. Wire read-only app-facing chat data after native auth transport posture is
   selected.
