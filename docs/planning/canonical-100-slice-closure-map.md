# Canonical 100-Slice Closure Map

## Purpose

This document turns the flagship UX/UI convergence lane into an execution map
with 100 bounded slices.

It exists to prevent random polishing and to keep work aligned with the
approved closure order:

1. public home
2. authenticated shell frame and sidebar exactness
3. dashboard
4. chat
5. personality
6. final cross-surface parity proof

The slices below are not equal in size.
They are intentionally small, visual, and closure-oriented.

## Governing Rules

- One surface at a time.
- Do not move to the next dependent surface until the current one is judged at
  least `95%` aligned with the active canonical spec.
- The active spec is the canonical image plus any explicit user-approved
  interpretation notes.
- If a surface reaches the parity gate earlier than the provisional slice
  budget, stop and move to the next surface instead of polishing forever.
- If a surface still visibly drifts after consuming its provisional slice
  budget, add a new bounded continuation lane instead of silently widening the
  current task.
