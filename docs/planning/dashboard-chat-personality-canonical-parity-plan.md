# Dashboard, Chat, And Personality Canonical Parity Plan

## Purpose

This plan tightens the current web UX convergence lane around the three
highest-value authenticated surfaces:

- dashboard-first shell framing
- `chat`
- `personality` preview

The approved canonical references already define the target quality bar.
This plan translates those images into one execution queue that preserves the
existing route topology while making the product feel much closer to the
approved flagship experience.

## Route Intent Clarification

The current route topology is now explicit and accepted:

- `dashboard` is now the first authenticated route and the flagship overview
- `chat` is the main product conversation route
- `personality` is still the richest embodied explanation route

This means dashboard parity in the current lane should be interpreted as:

- stronger flagship shell framing
- a dashboard hero stage that feels scenic, connected, and ceremonial
- richer route-entry posture
- shared premium structure that also uplifts `chat` and `personality`

## Current Implemented Baseline

- shared authenticated shell with left rail and desktop utility bar
- first-class `dashboard` route
- first-class `chat` route with transcript-first workspace and portrait panel
- first-class `personality` preview with embodied figure and anchored callouts
- route-specific raster artwork already exists for:
  - chat portrait background
  - dashboard intention card
  - dashboard summary band
  - personality figure

## Current Highest-Value Remaining Drift

### Shared Shell

- the top utility band still needs to feel more canonical and less generic
- the shell canvas should feel like one flagship environment, not a route host
- editorial spacing between shell chrome and route bodies still needs polish

### Dashboard

- the hero stage needs stronger connective detail between figure and signal
  cards
- the atmosphere around the figure needs more bespoke scenic depth
- the cognitive-flow band should read more like one instrument and less like a
  set of stacked cards

### Chat

- the route is directionally strong, but still needs more calm flagship
  cohesion from the shared shell
- the transcript area should remain primary while the surrounding atmosphere
  becomes more premium
- the route still needs final parity proof after deploy

### Personality

- the route now has the right structure, but still needs more visual
  connectedness between figure and callouts
- the hero stage should feel more ceremonial and less like panels sitting on a
  background
- side insight panels still need final material and spacing polish

## Current Gap Matrix

### Dashboard-First Shell Gap

What is missing:

- a proper utility top bar with search and utility actions
- a more flagship-feeling route canvas inside the authenticated shell
- stronger preview posture for deeper modules
- clearer distinction between shell framing and route-specific content

What should happen:

- the shell should inherit more of the canonical dashboard's polish,
  spaciousness, and premium utility rhythm
- shared chrome should feel intentional and luxurious without becoming noisy

### Chat Gap

What is already strong:

- conversation-first priority is mostly correct
- right-side support context is calmer than before
- composer and quick actions are closer to the target

What is still missing:

- shell utility chrome is still not as premium as the canonical chat target
- route framing still needs more flagship cohesion from the dashboard reference
- final screenshot parity must still prove the result at real breakpoints

### Personality Gap

What is still farthest from canonical:

- the hero is still too generic and too system-like
- the figure is not yet the central narrative object it should be
- anchored labels, symbolic artifacts, and planning board are underrepresented
- the top preview lacks the canonical split between:
  - figure-based embodied explanation
  - side insight panels
  - mind-layer timeline

What should happen:

- `personality` preview should become the clearest embodiment of AION's inner
  architecture in the whole app
- it should look like a true product showcase, not a dressed-up payload browser

## Required Shared Behaviors

### Shared Shell

- left rail remains calm and architectural
- desktop gets a premium utility bar
- route canvas spacing should feel continuous and editorial
- surface hierarchy should favor fewer, stronger panels over many equal boxes

### Shared Material System

- keep warm vellum surfaces
- keep restrained teal and copper accents
- reduce hard edges where a softer premium contour is possible
- use typography and spacing for hierarchy before extra ornament

### Shared Preview Language

- use richer route preview panels for flagship surfaces
- let `chat` stay calmer than `dashboard`
- let `personality` stay richer and more diagrammatic than `chat`

## Acceptance Conditions For This Lane

The lane is complete when:

- the authenticated shell carries clear flagship dashboard DNA
- `chat` feels calmer and more premium inside that shared shell
- `personality` preview visibly resembles the canonical embodied map target
- the route family reads as one product system
- parity evidence exists instead of subjective "close enough" impressions

## Remaining Risk

- polishing shell details without sufficiently upgrading the dashboard hero
  stage
- treating `chat` as done before post-deploy screenshot proof
- overbuilding dashboard motifs into `chat` instead of keeping it intimate
- flattening premium route atmosphere back into CSS-only gradients when a
  bespoke raster layer is the cleaner answer
