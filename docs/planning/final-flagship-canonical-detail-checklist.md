# Final Flagship Canonical Detail Checklist

## Purpose

This checklist translates the remaining visual drift between the current
authenticated web shell and the approved canonical flagship references into a
detail-level execution queue.

The target routes are:

- `dashboard`
- `chat`
- `personality`

The goal is no longer broad style convergence. The goal is last-mile parity:

- compositional rhythm
- surface hierarchy
- atmospheric balance
- mobile compression
- cross-route shell continuity

## Shared Shell Checklist

### Layout Spine

- [ ] Desktop rail width remains secondary to the flagship content canvas.
- [ ] Utility chrome feels inset and premium instead of product-admin.
- [ ] Route content begins with the true flagship surface rather than stacked
      intro chrome.
- [ ] Shared spacing between utility bar and route body is consistent across
      `dashboard`, `chat`, and `personality`.

### Material System

- [ ] Vellum surfaces remain warm and soft across all three routes.
- [ ] Teal and copper accents stay restrained and semantic.
- [ ] Rounded contours feel premium without making cards look inflated.
- [ ] Typography hierarchy does more work than decorative borders.

## Dashboard Checklist

### Hero Stage

- [ ] The center figure remains the strongest object on first read.
- [ ] Signal cards clearly point toward the figure rather than floating beside
      it.
- [ ] The hero atmosphere reads as one cognition scene instead of a background
      plus unrelated cards.
- [ ] Hero chips remain inside the flagship stage and do not escape into shell
      chrome.

### Guidance Column

- [ ] The first card reads as the lead recommendation.
- [ ] Supporting cards read as quieter companions, not equal-weight blocks.
- [ ] Recent activity feels embedded in the same editorial column.
- [ ] The `Today's intention` panel reads like a scenic closing note for the
      sidebar, not a separate promo card.

### Lower Dashboard Rhythm

- [ ] The lower trio of cards feels compact enough to keep the route close to
      a one-screen flagship read.
- [ ] `Active goals`, `Current focus`, and `Memory growth` each have a
      distinct role and material treatment.
- [ ] The lower grid does not compete with the hero stage.
- [ ] The final summary band closes the route instead of starting a second
      narrative.

### Mobile Dashboard

- [ ] The hero still dominates before the lower cards.
- [ ] Guidance and recent activity compress without turning into equal tiles.
- [ ] The scenic summary remains readable without excessive vertical drag.

## Chat Checklist

### Conversation Core

- [ ] Transcript remains the first and strongest surface on desktop.
- [ ] Starter transcript for empty history still feels premium and intentional.
- [ ] Top controls support the route quietly and never overshadow the messages.
- [ ] Composer, quick actions, and transcript read as one calm conversation
      instrument.

### Support Column

- [ ] Portrait panel sits high enough to be immediately useful.
- [ ] Planning overlay reads as one inset rather than a separate widget.
- [ ] Support cards feel like quiet context, not a second dashboard.
- [ ] Support-column spacing stays tighter than the transcript rhythm.

### Mobile Chat

- [ ] Portrait crop still feels premium when compressed.
- [ ] Planning overlay remains readable without covering the face awkwardly.
- [ ] Top controls wrap gracefully without becoming noisy.
- [ ] Composer and quick actions remain visually joined.

## Personality Checklist

### Embodied Preview

- [ ] Figure stage remains the ceremonial center of the route.
- [ ] Identity, knowledge, planning, skills, and role callouts are easy to
      locate at a glance.
- [ ] Connector lines and endpoints are visible enough to explain the system.
- [ ] The hero and the timeline feel like one embodied explanation surface.

### Side Column

- [ ] The side stack has clear hierarchy, not four equal panels.
- [ ] Highlights read as the anchor summary object.
- [ ] Conscious and subconscious sections stay distinct but visually related.
- [ ] Recent activity becomes the quietest support block in the stack.

### Mobile Personality

- [ ] The callouts remain legible after compression.
- [ ] Side panels stack in a calm editorial rhythm.
- [ ] The route still feels ceremonial instead of collapsing into generic cards.

## Execution Order

### Loop 1: Structural Rhythm

- dashboard guidance/recent/intention relationship
- dashboard lower-grid pacing
- chat support-column spacing and card hierarchy
- personality side-stack hierarchy and mobile compression

### Loop 2: Crop And Atmosphere

- dashboard hero crop and connector calibration
- chat portrait crop and overlay placement
- personality callout visibility and embodied-stage balance

### Loop 3: Final Proof

- deploy refreshed build
- capture desktop, tablet, and mobile screenshots
- compare each route directly with its canonical reference
- record the final residual mismatches, if any

## Completion Standard

This checklist is complete only when:

- all three flagship routes look structurally faithful to the canonical
  references
- the shared shell feels like one premium environment
- mobile compression preserves the same identity
- parity review finds only tiny crop-level differences instead of structural
  drift
